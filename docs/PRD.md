## 1. 개요 (Overview)

한국인 이주 준비자(유학생·취업자)가 **서울과 해외 도시의 생활비**를 항목별로 비교할 수 있는 모바일 앱. 사용자의 페르소나에 따라 맞춤화된 정보를 보여주고, 단순한 1:1 비교를 통해 "이 도시로 가면 얼마나 더 들까"라는 질문에 직관적으로 답한다.

**한 줄 정의:** _"서울 사는 한국인이, 가고 싶은 해외 도시의 생활비를 본인 기준으로 즉시 가늠할 수 있는 앱."_

---

## 2. 배경 및 문제 (Background & Problem)

- 한국인 유학·이주 준비 시 도시별 생활비 정보가 분산되어 있음 (블로그·카페·유튜브에 흩어진 단편 정보)
- 기존 도구(Numbeo, Expatistan 등)는 영어 기반이고 전 세계 평균 관점이라 **본국(서울) 대비 감각**이 잘 잡히지 않음
- 한국인 특화 정보 부족: **학비**, **비자/정착 비용**, **한국식 식재료(라면 등) 가격**
- 출시자 본인이 밴쿠버 유학 시절 마땅한 비교 도구가 없어 직접 겪은 불편이 출발점

---

## 3. 목표 / Non-목표 (Goals / Non-Goals)

### v1.0 목표

- 서울과 20개 해외 도시의 핵심 생활비 항목을 **1:1로 비교**
- **유학생/취업자** 페르소나 맞춤 정보 제공
- 데이터 **출처와 업데이트 일자**를 항상 투명하게 표시
- iOS·Android 양쪽 동시 출시

### Non-목표 (이번 버전에서 제외, 추후 검토)

- 사용자 계정·로그인·기기간 동기화
- 본국 = 서울 외 다른 도시 출발지
- 한국 이외 국적 사용자 지원
- 여러 도시 동시 비교(3-way 이상)
- 예산 시뮬레이터 모드
- 푸시 알림
- 사용자 커뮤니티/후기

## 4. 타겟 사용자 (Target Users)

**1차 타겟:** 한국인 이주 준비자

### 페르소나 A — 유학생

- 학부생/대학원생 또는 어학연수 준비자
- 핵심 질문: _"학비+생활비 다 합쳐서 월/년 얼마 들지?"_
- 관심 항목: **학비**, **셰어/기숙사 월세**, 식비, 교통, 비자
- 예산 민감도 높음, 셰어 거주 일반적

### 페르소나 B — 취업자

- 한국에서 경력 쌓은 후 해외 이직 또는 주재원
- 핵심 질문: _"실수령으로 서울 대비 얼마나 여유롭지?"_
- 관심 항목: **세금/실수령**, **1인용 월세**, 의료, 식비, 비자
- 가족 단위 가능성 있음

### 보조 옵션

- "아직 모름" — 둘 다 보여주는 기본 모드 (온보딩 부담 낮춤)

## 5. 사용자 시나리오 (User Scenarios)

### 시나리오 1: 첫 사용 (유학생)

1. 앱 설치·실행 → 온보딩에서 "유학생" 선택
2. 홈에서 "밴쿠버" 검색 → 비교 화면 진입
3. 상단 요약: "서울 175만 vs 밴쿠버 340만 (↑1.9배)"
4. 식비 카드 탭 → 우유·계란·신라면 등 식재료별 가격 차이 확인 → "신라면 2.5배 비싸네"
5. 즐겨찾기 추가
6. 토론토와도 비교 후 즐겨찾기

### 시나리오 2: 재방문 (취업자)

1. 앱 실행 → 즐겨찾기 카드에 "토론토 ↑1.7배" 미리 표시
2. 토론토 카드 탭 → 세금/실수령 카드 즉시 확인
3. 데이터 업데이트 일자(2026-04-01) 확인 → 신뢰

### 시나리오 3: 페르소나 전환

1. 처음에 "아직 모름"으로 시작했던 사용자
2. 진로 결정 후 설정에서 페르소나를 "취업자"로 변경
3. 다음 비교부터 학비 카드 사라지고 세금/실수령 카드 등장

## 6. 기능 요구사항 (Functional Requirements)

### F1. 온보딩

- 첫 실행 시 1회 표시 (`onboarded: true` 플래그)
- 환영 화면 → 페르소나 선택 (유학생 / 취업자 / 아직 모름)
- 선택 후 홈으로 이동
- 설정에서 언제든 페르소나 변경 가능

### F2. 홈

- **즐겨찾기 도시** 카드(가로 스크롤): 핵심 비교값 프리뷰 (예: "↑1.9배")
- **최근 본 도시** 리스트 (최대 5개, FIFO)
- **도시 검색**: 한글·영어 양방향
- **지역별 도시 리스트**: 북미 / 유럽 / 오세아니아 / 아시아 / 중동
- 첫 진입(즐겨찾기 0개) 시 안내 + "관심 도시 골라보기" CTA

### F3. 도시 비교 화면 (메인)

- 헤더: `서울 vs [도시명] [국기] · 즐겨찾기 토글 · 환율 정보`
- **상단 요약 카드**: "한 달 예상 총비용" — 서울값 / 도시값 / 배수
  - 평균 가정 기준임을 명시 (작은 캡션)
  - 우측 ❓ 아이콘 탭 → 가정값 설명 시트 (부록 C 참조)
  - 유학생 모드에서는 합계 아래 연간 학비를 별도 라인으로 참고 표시
- **카테고리 카드** (페르소나별 다름):
  - 공통: 월세, 식비, 교통, 비자/정착
  - 유학생 추가: 학비
  - 취업자 추가: 세금/실수령, 의료
- 각 카드 표시 형식: `서울값 → 도시값 (배수, 차액)`
- 차이 표시: **배수 + 절대 차액 동시** (% 사용 안 함)
- 색상 신호: 비싼 항목 빨강, 싼 항목 초록 (단, 색상에만 의존 안 함)
- 푸터: 출처 / 마지막 업데이트 일자

### F4. 항목 상세 화면

- 카테고리별 자유 구성 (공통 템플릿 강요 안 함)

| 카테고리       | 상세 구성                                                                                                                  |
| -------------- | -------------------------------------------------------------------------------------------------------------------------- |
| 🍴 식비        | 외식 2종(식당/카페) + **식재료 8~10종**(우유, 계란, 쌀, 닭가슴살, 빵, 양파, 사과, 신라면 등) + 월 예상 식비(외식형/자취형) |
| 🏠 월세        | 셰어/원룸/투룸/쓰리룸 × 시내/외곽 + 보증금·디포짓                                                                          |
| 🚇 교통        | 월정기권, 1회권, 택시 기본료, (취업자) 차량 유지비·보험                                                                    |
| 🎓 학비        | 주요 대학 3~5개 × 학사/석사 × 학기당·연간                                                                                  |
| 💼 세금/실수령 | 연봉대 2~3개(예: 6/8/10만 현지통화) × 실수령률                                                                             |
| 📋 비자/정착   | 비자 종류별 신청비 + 처리기간 + 부대비용                                                                                   |

- 각 상세 화면 푸터에 출처 링크

### F5. 설정

- 페르소나 변경
- 데이터 출처 보기 (별도 화면)
- 마지막 업데이트 일자 + 수동 새로고침
- 피드백 보내기 (이메일 링크)
- 앱 정보·버전

### F6. 데이터 갱신

- 앱 시작 시 원격(GitHub raw)에서 도시 JSON fetch
- AsyncStorage 24h TTL 캐시 (첫 실행은 번들된 시드)
- 사용자가 설정에서 수동 새로고침 가능
- 환율은 별도로 일 1회 fetch (open.er-api.com)

### F7. 즐겨찾기 / 최근 본 도시

- 모두 **로컬 저장** (AsyncStorage)
- 즐겨찾기 무제한 (실용상 ≤20개)
- 최근 본 도시는 자동 관리, 최대 5개

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 성능

- 콜드 스타트 ≤3초 (캐시 사용 시)
- 화면 전환 ≤300ms
- 데이터는 캐시 우선 → 즉시 표시 후 백그라운드 갱신

### 접근성

- 다이나믹 타입 지원 (시스템 글자 크기 반영)
- 색상에만 의존하지 않는 표기 (배수 숫자 + 화살표 + 색상)
- VoiceOver / TalkBack 기본 라벨

### 개인정보

- 계정·로그인 없음
- 모든 사용자 데이터는 로컬 저장
- 분석·추적 도구 없음 (v1.0)
- 개인정보 처리방침 페이지 (정적 텍스트)

### 데이터 정확성

- 모든 항목에 **출처 명시**
- **분기 1회** 정기 업데이트
- 가능한 항목은 복수 출처에서 교차 확인
- Numbeo·Expatistan 데이터 직접 복제 금지 (약관 준수)

### 지원 환경

- iOS 15+ / Android 8 (API 26)+
- 한국어 단일 (i18n 미적용)
- 네트워크 없을 때도 캐시 데이터로 동작

## 9. 기술 스택 (Tech Stack)

| 레이어        | 선택                            | 이유                                    |
| ------------- | ------------------------------- | --------------------------------------- |
| 언어          | TypeScript                      | 사용자 친숙, 타입 안정성                |
| 프레임워크    | **React Native (Expo Managed)** | 단일 코드베이스로 iOS+Android           |
| 라우팅        | Expo Router (파일 기반)         | Next.js 친화, 공식 권장                 |
| 스타일링      | NativeWind v4                   | Tailwind 클래스 그대로 사용             |
| 상태 관리     | Zustand                         | 보일러플레이트 적음, 5화면 규모에 충분  |
| 로컬 저장     | AsyncStorage                    | 페르소나·즐겨찾기·최근 본 도시          |
| 데이터 호스팅 | GitHub Public Repo (JSON)       | 무료, 버전 관리, 업데이트 간편          |
| 환율 API      | open.er-api.com                 | 무료, 키 불필요                         |
| 빌드/배포     | EAS Build + EAS Update          | OTA 업데이트로 코드 패치 시 재심사 회피 |

### 도입하지 않는 것 (의도적)

백엔드 서버, Redux, 인증/계정, 푸시 알림, Numbeo·Expatistan 유료 API, 다국어(i18n), 분석 도구

## 12. 위험 요소 / Open Questions (Risks & Open Questions)

### 위험

- **데이터 정확도:** 수동 큐레이션이라 누락·오차 가능 → 출처 투명성 + 사용자 제보 채널(v1.1)로 완화
- **데이터 갱신 부담:** 분기 1회도 도시 20개면 1인 부담 큼 → 항목 우선순위 정해 핵심부터 갱신
- **법적 리스크:** Numbeo·Expatistan 직접 사용 금지 엄수
- **앱 스토어 심사 거절 가능성:** 메타데이터·스크린샷·개인정보 처리방침 사전 준비
- **콜드 스타트:** 사용자 0명에서 시작 → 한인 유학·이주 커뮤니티에 시드 마케팅 필요 (마케팅 계획은 별도)

### 결정된 항목

- ✅ **항목별 신고 기능은 v1.0 제외, v1.1에 도입.** v1.0은 설정의 일반 피드백 이메일만 운영. 출시 일정 단축 우선.
- ✅ **"한 달 예상 총비용" 가정값 정의됨** — 부록 C 참조. 유학생은 학비 별도 표시, 취업자는 의료비 v1.0 제외.

### Open Questions

- v1.0 일반 피드백 이메일 주소는 어떤 걸로? (개인 이메일 vs 전용 주소 신설)
- 데이터 업데이트 시 변경 이력(changelog)을 사용자에게 보여줄지?
- 출시 시 한국 외 사용자 접근(예: 영어권 한국 유학생)은 차단? 허용?

## 부록 C. "한 달 예상 총비용" 계산 정의

요약 카드 합계는 아래 가정에 따라 계산되며, 사용자에게 ❓ 아이콘으로 투명하게 안내한다.

### C.1 유학생 합계 (월)

| 항목 | 가정                          |
| ---- | ----------------------------- |
| 월세 | 셰어 1실                      |
| 식비 | 자취 70% + 외식 30% 혼합 평균 |
| 교통 | 월정기권                      |

→ 합계 아래에 **연간 학비**를 참고용 별도 라인으로 표시 (월 합계엔 미포함).

### C.2 취업자 합계 (월)

| 항목 | 가정                          |
| ---- | ----------------------------- |
| 월세 | 1인 원룸                      |
| 식비 | 자취 50% + 외식 50% 혼합 평균 |
| 교통 | 월정기권                      |

→ 의료비는 v1.0 월 합계에서 제외. 별도 카드로 도시별 보험료 안내는 가능 (데이터 확보된 경우에 한해).

### C.3 모든 페르소나 공통 — 월 합계에 **미포함**

- 통신비
- 의류·여가·문화비
- 1회성 비용 (비자, 정착, 디포짓)
- 의료비 (v1.0)
- 세금 (지출이 아닌 수입 차감 항목 — 별도 카드에서 실수령으로 표기)
- 학비 (유학생) — 위 별도 라인 참고

### C.4 환율

- 모든 외화 항목을 KRW로 변환 후 합산
- 환율은 fetch 시점 기준 (open.er-api.com)
- 합계 카드 캡션에 "환율 기준일: YYYY-MM-DD" 표시
